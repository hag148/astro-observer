# 天文观测系统 Pro - 项目梳理文档

## 1. 用户场景

### 1.1 谁在用？
- **业余天文爱好者**：使用系统进行月球、行星、恒星和深空天体的观测与摄影，提升观测体验和图像质量。
- **天文摄影初学者**：通过系统的易用性和专业功能，快速入门天文摄影，学习图像处理技巧。
- **教育机构的天文教学**：用于课堂教学，展示天文现象，培养学生对天文学的兴趣。
- **科学研究辅助观测**：为小型科研项目提供观测数据和图像处理支持。

### 1.2 什么情况下用？
- **夜间天文观测**：在晴朗的夜晚，使用望远镜配合USB摄像头进行天体观测。
- **天文摄影**：拍摄月球表面细节、行星特征、恒星和深空天体。
- **图像处理**：对拍摄的天文图像进行增强、叠加和分析。
- **观测记录**：记录观测过程和结果，生成观测日志。
- **教学演示**：在课堂上展示天文观测过程和结果。

## 2. 业务流程

### 2.1 前置条件
- **硬件准备**：
  - 计算机（CPU: Intel i5 或同等性能，内存: 4GB 以上，存储: 至少 1GB 可用空间，USB 3.0 接口）
  - USB 摄像头（支持高分辨率）
  - 望远镜（可选，用于放大观测目标）
  - 稳定的电源供应
- **软件准备**：
  - Java 17 或更高版本
  - 天文观测系统 Pro 软件
- **环境准备**：
  - 晴朗的天气
  - 远离光污染的观测地点
  - 稳定的观测平台

### 2.2 正常流程
1. **启动系统**：运行天文观测系统 Pro 软件。
2. **摄像头连接**：
   - 系统自动检测并列出可用的 USB 摄像头。
   - 用户选择要使用的摄像头。
   - 系统初始化摄像头并显示实时预览。
3. **参数配置**：
   - 用户调整摄像头分辨率、曝光时间等参数。
   - 选择天文观测预设（如月球、行星、深空）。
4. **观测目标**：
   - 调整望远镜指向目标天体。
   - 系统显示实时预览，用户微调望远镜位置。
5. **图像采集**：
   - 单帧捕获：拍摄单张图像。
   - 连续拍摄：设置拍摄帧数和间隔，系统自动连续拍摄。
6. **图像处理**：
   - 对捕获的图像进行增强处理（亮度、对比度、锐化等）。
   - 对多张图像进行叠加处理，提高信噪比。
7. **天体检测**：
   - 系统自动识别图像中的星点、行星等天体。
   - 标记天体位置并显示相关信息。
8. **观测记录**：
   - 系统记录观测会话信息。
   - 用户添加观测笔记。
9. **数据导出**：
   - 导出处理后的图像。
   - 导出观测日志。
10. **系统关闭**：
    - 系统自动释放摄像头资源。
    - 保存会话数据。

### 2.3 异常流程
1. **摄像头连接失败**：
   - **现象**：系统无法检测到摄像头或连接失败。
   - **处理**：
     - 检查摄像头是否正确连接。
     - 检查 USB 驱动是否正常。
     - 尝试更换 USB 端口。
     - 重启系统。
2. **图像质量问题**：
   - **现象**：图像模糊、过暗、过亮或有噪点。
   - **处理**：
     - 调整摄像头参数（曝光时间、增益等）。
     - 检查望远镜对焦。
     - 使用图像处理功能进行增强。
3. **系统崩溃**：
   - **现象**：系统突然关闭或无响应。
   - **处理**：
     - 重启系统。
     - 检查系统日志获取错误信息。
     - 降低摄像头分辨率或帧率。
4. **存储不足**：
   - **现象**：系统提示存储空间不足。
   - **处理**：
     - 清理磁盘空间。
     - 调整图像保存格式或分辨率。
     - 导出已捕获的图像到外部存储。
5. **观测环境问题**：
   - **现象**：天气突然变化（如云层遮挡）。
   - **处理**：
     - 暂停观测。
     - 保存当前会话数据。
     - 等待天气好转或结束观测。

## 3. 技术实现

### 3.1 接口设计

#### 3.1.1 摄像头管理接口
- **CameraManager 接口**：
  - `List<Camera> getAvailableCameras()`：获取可用摄像头列表。
  - `void connectCamera(Camera camera)`：连接指定摄像头。
  - `void disconnectCamera()`：断开当前摄像头连接。
  - `void setResolution(Resolution resolution)`：设置摄像头分辨率。
  - `void setExposure(int exposure)`：设置曝光时间。
  - `BufferedImage captureFrame()`：捕获当前帧。
  - `void startPreview()`：开始预览。
  - `void stopPreview()`：停止预览。

#### 3.1.2 图像处理接口
- **ImageProcessor 接口**：
  - `BufferedImage enhanceImage(BufferedImage image, EnhancementParams params)`：增强图像。
  - `List<DetectedObject> detectObjects(BufferedImage image)`：检测图像中的天体。
  - `BufferedImage applyFilter(BufferedImage image, FilterType filter)`：应用滤镜。

- **ImageStacker 接口**：
  - `void addImage(BufferedImage image)`：添加图像到堆叠队列。
  - `BufferedImage stackImages(StackingMethod method)`：执行图像叠加。
  - `double calculateSNR()`：计算信噪比。
  - `void clear()`：清空堆叠队列。

#### 3.1.3 观测会话接口
- **ObservationSession 接口**：
  - `void startSession()`：开始观测会话。
  - `void endSession()`：结束观测会话。
  - `void addImage(BufferedImage image, String metadata)`：添加图像到会话。
  - `void addNote(String note)`：添加观测笔记。
  - `void saveSession(String path)`：保存会话数据。
  - `void loadSession(String path)`：加载会话数据。

#### 3.1.4 GUI 接口
- **AdvancedAstronomyFrame**：
  - `void initializeUI()`：初始化用户界面。
  - `void loadCameras()`：加载可用摄像头。
  - `void setupEventHandlers()`：设置事件处理器。
  - `void updatePreview(BufferedImage image)`：更新预览图像。
  - `void showMessage(String message)`：显示消息。

### 3.2 数据模型

#### 3.2.1 核心数据模型
- **Camera**：
  - 字段：`id` (String), `name` (String), `resolutions` (List<Resolution>), `connected` (boolean)
  - 用途：表示摄像头设备信息

- **Resolution**：
  - 字段：`width` (int), `height` (int)
  - 用途：表示摄像头分辨率

- **ObservationSession**：
  - 字段：`id` (String), `startTime` (LocalDateTime), `endTime` (LocalDateTime), `images` (List<ObservationImage>), `notes` (List<String>), `cameraInfo` (String)
  - 用途：表示观测会话数据

- **ObservationImage**：
  - 字段：`id` (String), `timestamp` (LocalDateTime), `imagePath` (String), `metadata` (Map<String, Object>), `detectedObjects` (List<DetectedObject>)
  - 用途：表示观测图像数据

- **DetectedObject**：
  - 字段：`id` (String), `type` (String), `x` (int), `y` (int), `size` (int), `brightness` (double), `confidence` (double)
  - 用途：表示检测到的天体信息

- **EnhancementParams**：
  - 字段：`brightness` (double), `contrast` (double), `saturation` (double), `sharpness` (double), `denoise` (double)
  - 用途：表示图像增强参数

## 4. 质量保障

### 4.1 测试要点

#### 4.1.1 功能测试
- **摄像头管理**：
  - 测试摄像头检测功能，确保能正确识别连接的摄像头。
  - 测试摄像头切换功能，确保能在多个摄像头之间切换。
  - 测试分辨率调整功能，确保能正确设置不同分辨率。
  - 测试实时预览功能，确保预览流畅。

- **图像采集**：
  - 测试单帧捕获功能，确保能正确保存图像。
  - 测试连续拍摄功能，确保能按设置的参数连续拍摄。
  - 测试不同格式的图像保存功能。

- **图像处理**：
  - 测试图像增强功能，确保能正确应用增强参数。
  - 测试图像叠加功能，确保能正确叠加多张图像。
  - 测试不同叠加算法的效果。

- **天体检测**：
  - 测试星点识别功能，确保能正确识别图像中的星点。
  - 测试行星检测功能，确保能正确识别行星。
  - 测试位置标记功能，确保能正确标记天体位置。

- **观测管理**：
  - 测试观测会话记录功能，确保能正确记录会话信息。
  - 测试图像分类存储功能，确保能按分类存储图像。
  - 测试数据导出功能，确保能正确导出数据。

#### 4.1.2 性能测试
- **实时预览性能**：测试不同分辨率下的预览帧率，确保流畅运行。
- **图像处理性能**：测试图像增强和叠加的处理时间，确保在可接受范围内。
- **内存使用**：测试系统在长时间运行时的内存使用情况，确保无内存泄漏。
- **CPU 使用率**：测试系统在处理图像时的 CPU 使用率，确保系统稳定运行。

#### 4.1.3 兼容性测试
- **操作系统兼容性**：测试系统在 Windows、Linux、macOS 等不同操作系统上的运行情况。
- **摄像头兼容性**：测试系统与不同品牌、型号的 USB 摄像头的兼容性。
- **分辨率兼容性**：测试系统对不同分辨率摄像头的支持情况。

#### 4.1.4 可靠性测试
- **异常处理**：测试系统在遇到异常情况时的处理能力，如摄像头断开连接、存储不足等。
- **稳定性**：测试系统在长时间运行时的稳定性，确保无崩溃或无响应情况。
- **资源释放**：测试系统在关闭时是否正确释放资源，如摄像头、内存等。

### 4.2 监控指标

#### 4.2.1 系统监控
- **CPU 使用率**：监控系统在运行过程中的 CPU 使用率，确保不超过 80%。
- **内存使用**：监控系统的内存使用情况，确保无内存泄漏。
- **磁盘空间**：监控磁盘空间使用情况，确保有足够的存储空间。
- **摄像头状态**：监控摄像头的连接状态，确保摄像头正常工作。

#### 4.2.2 性能监控
- **预览帧率**：监控实时预览的帧率，确保不低于 15fps。
- **图像处理时间**：监控图像处理的时间，确保不超过 1 秒。
- **图像叠加时间**：监控图像叠加的时间，确保在可接受范围内。
- **天体检测时间**：监控天体检测的时间，确保不超过 100ms。

#### 4.2.3 错误监控
- **错误率**：监控系统的错误率，确保错误率低于 1%。
- **异常类型**：监控系统出现的异常类型，分析异常原因。
- **错误日志**：定期检查错误日志，及时发现和解决问题。

### 4.3 应急预案

#### 4.3.1 系统崩溃应急预案
- **症状**：系统突然关闭或无响应。
- **应急措施**：
  - 重启系统。
  - 检查系统日志，分析崩溃原因。
  - 降低摄像头分辨率或帧率。
  - 如果问题持续，恢复到之前的稳定版本。

#### 4.3.2 摄像头故障应急预案
- **症状**：摄像头连接失败或无图像。
- **应急措施**：
  - 检查摄像头连接。
  - 重启摄像头。
  - 尝试使用备用摄像头。
  - 更新摄像头驱动。

#### 4.3.3 存储不足应急预案
- **症状**：系统提示存储空间不足。
- **应急措施**：
  - 清理磁盘空间。
  - 导出已捕获的图像到外部存储。
  - 调整图像保存格式或分辨率。
  - 暂停拍摄，直到有足够的存储空间。

#### 4.3.4 网络连接故障应急预案（针对未来云端功能）
- **症状**：无法连接到云端服务。
- **应急措施**：
  - 检查网络连接。
  - 切换到离线模式。
  - 本地保存数据，待网络恢复后同步。

#### 4.3.5 数据丢失应急预案
- **症状**：观测数据丢失。
- **应急措施**：
  - 检查自动保存功能是否开启。
  - 恢复最近的备份。
  - 检查临时文件目录，寻找未保存的文件。
  - 实施定期自动备份策略。

## 5. 总结

天文观测系统 Pro 是一个专为业余天文学者和摄影爱好者设计的专业天文观测应用程序。系统通过普通 USB 摄像头配合望远镜，实现月球、行星、恒星和深空天体的观测与摄影。

系统的核心功能包括摄像头管理、图像采集、图像处理、图像叠加、天体检测和观测管理。技术上采用 Java 17+ 作为核心开发语言，结合 OpenCV 进行图像处理，WebCam Capture API 进行相机管理，Swing 配合 FlatLaf 实现现代化的 GUI 界面。

通过本项目梳理文档，我们详细分析了系统的用户场景、业务流程、技术实现和质量保障措施，为系统的开发、测试和维护提供了全面的指导。
