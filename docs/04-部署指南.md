# 天文观测系统 Pro - 部署指南

## 1. 环境准备

### 1.1 系统要求

#### 最低配置
- **操作系统**: Windows 10/11, Linux (Ubuntu 20.04+), macOS 11+
- **Java**: JDK 17 或更高版本
- **内存**: 4GB RAM
- **存储**: 1GB 可用空间
- **USB**: USB 2.0 或更高

#### 推荐配置
- **操作系统**: Windows 11, Ubuntu 22.04+, macOS 13+
- **Java**: JDK 17 或 21
- **内存**: 8GB+ RAM
- **存储**: 10GB+ SSD
- **USB**: USB 3.0 或更高
- **CPU**: Intel i5/AMD Ryzen 5 或更高

### 1.2 必需软件

## 3. 运行程序

### 3.1 Windows 部署

#### 使用启动脚本
```bash
# 双击运行
run.bat

# 或在命令行中运行
run.bat
```

#### 直接运行 JAR
```bash
java -jar target\astro-observer-1.0.0.jar
```

#### 创建桌面快捷方式

1. 右键桌面 → 新建 → 快捷方式
2. 位置: `java -jar "C:\Path\To\astro-observer\target\astro-observer-1.0.0.jar"`
3. 命名: "天文观测系统"
4. 更改图标（可选）

### 3.2 Linux 部署

#### 使用启动脚本
```bash
# 赋予执行权限
chmod +x run.sh

# 运行
./run.sh
```

#### 直接运行 JAR
```bash
java -jar target/astro-observer-1.0.0.jar
```

#### 创建桌面启动器

创建 `~/.local/share/applications/astro-observer.desktop`:
```ini
[Desktop Entry]
Version=1.0
Type=Application
Name=天文观测系统 Pro
Comment=Astronomy Observation System
Exec=java -jar /path/to/astro-observer/target/astro-observer-1.0.0.jar
Icon=/path/to/icon.png
Terminal=false
Categories=Science;
```

### 3.3 macOS 部署

#### 使用启动脚本
```bash
# 赋予执行权限
chmod +x run.sh

# 运行
./run.sh
```

#### 创建应用程序包

1. 创建应用目录结构:
```bash
mkdir -p AstroObserver.app/Contents/MacOS
mkdir -p AstroObserver.app/Contents/Resources
```

2. 创建 Info.plist:
```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>CFBundleExecutable</key>
    <string>launch</string>
    <key>CFBundleName</key>
    <string>AstroObserver</string>
    <key>CFBundleVersion</key>
    <string>1.0.0</string>
    <key>CFBundlePackageType</key>
    <string>APPL</string>
</dict>
</plist>
```

3. 创建启动脚本:
```bash
#!/bin/bash
cd /Applications/AstroObserver.app/Contents/Resources
java -jar astro-observer-1.0.0.jar
```

## 4. 目录结构

### 4.1 运行时目录结构

```
astro-observer/
├── target/
│   └── astro-observer-1.0.0.jar    # 可执行 JAR
├── captures/                        # 图像存储目录
├── sessions/                       # 会话记录目录
```

### 4.2 数据目录说明

| 目录 | 用途 | 说明 |
|------|------|------|
| captures/ | 图像存储 | 保存捕获的图像文件 |
| sessions/ | 会话记录 | 保存观测会话信息 |
| logs/ | 日志文件 | 应用运行日志 |
| config/ | 配置文件 | 用户配置（可选） |

## 5. 配置管理

### 5.1 JVM 参数调优

#### Windows
编辑 `run.bat`，在 `java` 命令后添加参数:
```batch
java -Xms512m -Xmx2048m -jar "target\astro-observer-1.0.0.jar"
```

#### Linux/Mac
编辑 `run.sh`，在 `java` 命令后添加参数:
```bash
java -Xms512m -Xmx2048m -jar target/astro-observer-1.0.0.jar
```

#### 常用 JVM 参数

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| -Xms | 初始堆大小 | 512m |
| -Xmx | 最大堆大小 | 2048m |
| -XX:NewRatio | 新生代与老年代比例 | 2 |
| -XX:+UseG1GC | 使用 G1 垃圾收集器 | - |

### 5.2 应用配置

创建 `config/application.properties`:
```properties
# 摄像头配置
camera.default.index=0
camera.default.resolution=1920x1080
camera.fps=30

# 图像处理配置
processing.enhance.enabled=true
processing.detection.threshold=50
processing.stack.maxSize=100

# 文件存储配置
storage.captures.dir=captures
storage.sessions.dir=sessions
storage.maxSize=10GB

# 日志配置
logging.level=INFO
logging.file.path=logs/app.log
logging.maxSize=10MB
logging.maxFiles=10
```

## 6. 故障排查

### 6.1 摄像头问题

#### 问题: 摄像头未识别
**排查步骤**:
1. 检查 USB 连接
2. 确认驱动已安装
3. 测试系统摄像头应用
4. 尝试重新插拔

**Linux 额外检查**:
```bash
# 检查摄像头设备
ls /dev/video*
# 检查权限
sudo chmod 666 /dev/video0
```

#### 问题: 分辨率设置失败
**解决方案**:
1. 查询支持分辨率
2. 选择支持的分辨率
3. 重启摄像头

### 6.2 性能问题

#### 问题: 帧率过低
**优化方案**:
1. 降低分辨率
2. 关闭后台应用
3. 增加 JVM 堆内存
4. 使用 USB 3.0 接口

#### 问题: 内存不足
**解决方案**:
1. 减少图像堆栈大小
2. 增加 JVM 堆内存
3. 定期清理 captures 目录

### 6.3 运行时错误

#### 问题: OpenCV 库加载失败
**解决方案**:
```bash
# 检查 OpenCV 库
java -Djava.library.path=/path/to/opencv/lib -jar target/astro-observer-1.0.0.jar
```

#### 问题: 端口占用
**解决方案**:
```bash
# Windows
netstat -ano | findstr :8080
taskkill /PID <PID> /F

# Linux/Mac
lsof -ti:8080 | xargs kill -9
```
