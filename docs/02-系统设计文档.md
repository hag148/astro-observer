# 天文观测系统 Pro - 系统设计文档

## 1. 设计原则

### 1.1 SOLID 原则
- **单一职责原则 (SRP)**: 每个类只负责一项功能
- **开闭原则 (OCP)**: 对扩展开放，对修改关闭
- **里氏替换原则 (LSP)**: 子类可替换父类
- **接口隔离原则 (ISP)**: 接口细粒度，不强迫依赖不需要的方法
- **依赖倒置原则 (DIP)**: 依赖抽象而非具体实现

### 1.2 设计模式应用
- **观察者模式**: 摄像头帧监听
- **工厂模式**: 摄像头配置创建
- **策略模式**: 图像叠加算法
- **单例模式**: OpenCV 库加载

### 2.3 图像叠加

#### ImageStacker
```java
public class ImageStacker {
    private List<Mat> stackedImages;
    private int maxStackSize;

    // 叠加方法
    public Mat stack()                    // 平均叠加
    public Mat medianStack()              // 中位数叠加
    public Mat kappaSigmaStack(double kappa, double sigma)

    // 辅助方法
    public double calculateSNR(Mat image)
}
```

**叠加算法对比**:

| 算法 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 平均叠加 | 速度快，简单 | 受异常值影响 | 一般情况 |
| 中位数叠加 | 去除异常值 | 计算较慢 | 有干扰源时 |
| Kappa-Sigma | 最佳质量 | 计算复杂 | 高质量需求 |

**Kappa-Sigma 算法流程**:
```
1. 计算所有像素的平均值 μ 和标准差 σ
2. 对于每个像素:
   if |pixel - μ| > κ·σ:
       丢弃该像素（异常值）
   else:
       保留该像素
3. 对保留的像素求平均
```

## 3. 数据流设计

### 3.1 摄像头数据流
```
USB Camera → WebCam Capture API → CameraManager → FrameListener → GUI Display
```

### 3.2 图像处理数据流
```
BufferedImage (GUI) → Mat (OpenCV) → 算法处理 → Mat (处理结果) → BufferedImage (GUI)
```

### 3.3 图像叠加数据流
```
多帧图像 → ImageStacker → 叠加算法 → 叠加结果 → 信噪比计算
```

## 5. 异常处理

### 5.1 异常分类

| 异常类型 | 处理方式 | 用户体验 |
|----------|----------|----------|
| 摄像头未连接 | 检测提示 | 显示警告 |
| 分辨率不支持 | 回退到默认 | 自动调整 |
| 内存不足 | 释放资源 | 提示用户 |
| 图像处理失败 | 跳过当前帧 | 继续运行 |
| 保存失败 | 重试或提示 | 显示错误 |

### 5.2 资源管理

```java
try {
    // 使用资源
    Mat mat = new Mat();
    // 处理逻辑
} finally {
    // 释放资源
    mat.release();
}
```

## 6. 性能优化

### 6.1 多线程处理
- 使用 SwingWorker 进行后台处理
- 避免阻塞 UI 线程
- 合理的线程池管理

### 6.2 内存管理
- 及时释放 OpenCV Mat 对象
- 限制图像堆栈大小
- 软引用缓存

### 6.3 图像处理优化
- 使用 ROI (Region of Interest)
- 降低中间处理分辨率
- 批量处理减少重复计算
