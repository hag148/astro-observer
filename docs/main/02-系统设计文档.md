# 天文观测系统 Pro - 系统设计文档

## 1. 设计原则

### 1.1 SOLID 原则
- **单一职责原则 (SRP)**: 每个类只负责一项功能
- **开闭原则 (OCP)**: 对扩展开放，对修改关闭
- **里氏替换原则 (LSP)**: 子类可替换父类
- **接口隔离原则 (ISP)**: 接口细粒度，不强迫依赖不需要的方法
- **依赖倒置原则 (DIP)**: 依赖抽象而非具体实现

### 1.2 设计模式应用
- **观察者模式**: 摄像头帧监听
- **工厂模式**: 摄像头配置创建
- **策略模式**: 图像叠加算法
- **单例模式**: OpenCV 库加载

### 2.3 图像叠加

#### ImageStacker
```java
public class ImageStacker {
    private List<Mat> stackedImages;
    private int maxStackSize;

    // 叠加方法
    public Mat stack()                    // 平均叠加
    public Mat medianStack()              // 中位数叠加
    public Mat kappaSigmaStack(double kappa, double sigma)

    // 辅助方法
    public double calculateSNR(Mat image)
}
```

**叠加算法对比**:

| 算法 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 平均叠加 | 速度快，简单 | 受异常值影响 | 一般情况 |
| 中位数叠加 | 去除异常值 | 计算较慢 | 有干扰源时 |
| Kappa-Sigma | 最佳质量 | 计算复杂 | 高质量需求 |

**Kappa-Sigma 算法流程**:
```
1. 计算所有像素的平均值 μ 和标准差 σ
2. 对于每个像素:
   if |pixel - μ| > κ·σ:
       丢弃该像素（异常值）
   else:
       保留该像素
3. 对保留的像素求平均
```

### 6.4 渲染性能优化

#### 6.4.1 Graphics2D AffineTransform

**问题**：
- 传统图像缩放需要创建新的BufferedImage
- 高倍缩放（如10倍）会产生100倍像素的图像
- 导致内存占用高和渲染卡顿

**解决方案**：
```java
// 使用变换而非缩放
Graphics2D g2d = (Graphics2D) g.create();
g2d.translate(labelWidth / 2, labelHeight / 2);
g2d.scale(zoomLevel, zoomLevel);
g2d.translate(-imageWidth / 2, -imageHeight / 2);
g2d.translate(-viewOffset.x, -viewOffset.y);
g2d.drawImage(originalImage, 0, 0, null); // 始终绘制原始图像
```

**优势**：
- 内存使用恒定，不受缩放倍数影响
- 渲染性能稳定，支持流畅缩放
- 支持任意倍数缩放（0.1x - 20x）

#### 6.4.2 坐标变换

**屏幕坐标到图像坐标**：
```java
Point2D screenToImage(double screenX, double screenY) {
    double centerX = labelWidth / 2;
    double centerY = labelHeight / 2;
    double imageX = (screenX - centerX) / zoomLevel + imageWidth / 2 + viewOffset.x;
    double imageY = (screenY - centerY) / zoomLevel + imageHeight / 2 + viewOffset.y;
    return new Point2D.Double(imageX, imageY);
}
```

**应用场景**：
- 点击放大定位
- 天体标记显示
- 拖拽移动计算

#### 6.4.3 渲染优化技巧

**RenderingHints配置**：
```java
g2d.setRenderingHint(RenderingHints.KEY_INTERPOLATION,
                     RenderingHints.VALUE_INTERPOLATION_BILINEAR);
g2d.setRenderingHint(RenderingHints.KEY_RENDERING,
                     RenderingHints.VALUE_RENDER_QUALITY);
g2d.setRenderingHint(RenderingHints.KEY_ANTIALIASING,
                     RenderingHints.VALUE_ANTIALIAS_ON);
```

**性能指标**：
- 缩放响应时间：< 16ms (60fps)
- 拖拽延迟：< 10ms
- 内存占用：~10MB（与缩放无关）

#### 7.2.2 标记绘制流程

```java
drawMarker(Graphics2D g2d, DetectedObject obj) {
    // 1. 转换坐标
    Point2D screenPos = imageToScreen(obj.center);

    // 2. 绘制黄色圆圈
    g2d.setColor(Color.YELLOW);
    Ellipse2D circle = new Ellipse2D.Double(
        screenPos.getX() - radius * zoomLevel,
        screenPos.getY() - radius * zoomLevel,
        radius * 2 * zoomLevel,
        radius * 2 * zoomLevel
    );
    g2d.draw(circle);

    // 3. 绘制红色十字准星
    g2d.setColor(Color.RED);
    g2d.drawLine(screenPos.getX() - 10, screenPos.getY(),
                 screenPos.getX() + 10, screenPos.getY());
    g2d.drawLine(screenPos.getX(), screenPos.getY() - 10,
                 screenPos.getX(), screenPos.getY() + 10);

    // 4. 绘制绿色标签
    g2d.setColor(Color.GREEN);
    g2d.drawString(obj.name, screenPos.getX() + 15, screenPos.getY() - 10);
}
```
