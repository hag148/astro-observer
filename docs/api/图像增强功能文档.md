# 图像增强功能文档

## 概述

天文观测系统 Pro 的图像增强功能提供9种专业的天文图像处理算法，专门针对不同类型的天体观测进行优化。

## 增强算法列表

### 1. BASIC - 基础增强

**组合算法**：直方图均衡化 + 锐化

**适用场景**：
- 通用天文图像增强
- 快速提升图像对比度

**技术实现**：
- 首先使用直方图均衡化增强整体对比度
- 然后应用锐化算法突出细节

### 2. CLAHE - 自适应直方图均衡化

**算法类型**：局部直方图均衡化

**适用场景**：
- 需要保持局部细节的图像
- 避免过度增强噪声的图像

**技术参数**：
- 限制对比度：2.0
- 2x2网格
- 5x5直方图bins

**优点**：
- 保留局部细节
- 避免过度增强
- 适合低光环境

### 3. UNSHARP_MASK - 反锐化掩码

**算法类型**：基于掩码的锐化

**适用场景**：
- 需要增强边缘和纹理
- 提高图像清晰度

**技术实现**：
- 创建图像的模糊版本
- 从原图中减去模糊版本
- 将结果与原图相加

**参数**：
- 强度：1.5
- 半径：1.0
- 阈值：0

### 4. MORPHOLOGY - 形态学增强

**算法类型**：形态学操作

**适用场景**：
- 增强图像结构
- 平滑噪声
- 突出特定特征

**技术实现**：
- 开运算：去除小噪声
- 闭运算：填充小孔洞
- 形态梯度：增强边缘

### 5. DENOISE - 降噪增强

**算法类型**：双边滤波

**适用场景**：
- 减少图像噪声
- 保持边缘清晰

**技术参数**：
- 滤波半径：10
- 颜色空间半径：10
- 色彩 sigma：75
- 空间 sigma：75

**优点**：
- 保边降噪
- 适合天文图像
- 保持细节

### 6. LUCY_RICHARDSON - Lucy-Richardson反卷积

**算法类型**：迭代反卷积

**适用场景**：
- 模糊图像恢复
- 提高图像清晰度

**技术实现**：
- 迭代次数：5
- 正则化：0.01

**注意事项**：
- 可能会放大噪声
- 需要适当调整参数

### 7. STRETCHING - 线性拉伸

**算法类型**：线性对比度拉伸

**适用场景**：
- 低对比度图像
- 快速增强

**技术实现**：
- 自动计算图像直方图
- 将像素值拉伸到[0, 255]范围

**优点**：
- 简单快速
- 保持图像比例

### 8. GAMMA_CORRECTION - Gamma校正

**算法类型**：非线性亮度调整

**适用场景**：
- 调整图像亮度曲线
- 平衡高光和阴影

**技术参数**：
- Gamma值：2.2（标准值）

**效果**：
- Gamma < 1: 增强高光
- Gamma > 1: 增强阴影

### 9. COMBINED - 综合增强

**算法类型**：多算法组合

**适用场景**：
- 复杂图像处理需求
- 获得最佳增强效果

**组合策略**：
- 先进行降噪处理
- 然后应用CLAHE增强
- 最后进行锐化处理

**效果**：
- 平衡噪声和细节
- 提升整体质量

## 专用增强模式

### 星点增强

**目标**：专门针对星星观测

**适用算法**：
- CLAHE（主要）
- Gamma校正
- 锐化

**优化参数**：
- 高对比度增强
- 保留星点锐度
- 避免过度增强背景

**效果**：
- 星点更清晰
- 背景更暗
- 对比度提升

### 深空天体增强

**目标**：专门针对星云、星系

**适用算法**：
- CLAHE
- 双边滤波
- 反锐化掩码

**优化参数**：
- 保持细节
- 平滑背景
- 增强颜色

**效果**：
- 星云纹理更清晰
- 颜色更鲜艳
- 背景更干净

### 行星表面增强

**目标**：专门针对月球、行星

**适用算法**：
- 锐化
- Gamma校正
- 对比度调整

**优化参数**：
- 突出表面细节
- 平衡亮度和对比度
- 保持真实颜色

**效果**：
- 表面纹理清晰
- 颜色准确
- 对比度适宜

## 使用方法

### 基础使用

```java
// 创建图像处理器
ImageProcessor processor = new ImageProcessor();

// 加载图像
BufferedImage image = processor.loadImage("capture.jpg");

// 应用增强
BufferedImage enhanced = processor.enhanceForAstronomy(image, "CLAHE");

// 保存结果
processor.saveImage(enhanced, "enhanced.jpg");
```

### 使用专用模式

```java
// 星点增强
BufferedImage starEnhanced = processor.enhanceForAstronomy(
    image,
    "STAR_ENHANCEMENT"
);

// 深空天体增强
BufferedImage deepSpaceEnhanced = processor.enhanceForAstronomy(
    image,
    "DEEP_SPACE_ENHANCEMENT"
);

// 行星表面增强
BufferedImage planetEnhanced = processor.enhanceForAstronomy(
    image,
    "PLANET_SURFACE_ENHANCEMENT"
);
```

### 控制面板使用

GUI界面提供完整的控制面板：

1. **算法选择**：下拉菜单选择9种算法之一
2. **专用模式**：星点/深空/行星快速增强
3. **参数调整**：
   - 强度滑块
   - 半径滑块
   - 阈值滑块
4. **说明显示**：当前算法的详细说明
5. **快捷键**：按 `E` 快速应用增强

### 算法实现

所有算法均使用 OpenCV 实现：

- **CLAHE**: `opencv_imgproc.createCLAHE()`
- **锐化**: `opencv_imgproc.filter2D()`
- **形态学**: `opencv_imgproc.morphologyEx()`

- **反卷积**: 自定义实现

## 常见问题

### Q: 哪种算法最适合星星观测？

A: 推荐使用 **星点增强** 模式或 **CLAHE** 算法。CLAHE能提供最佳的对比度提升，同时保持星点的锐度。

### Q: 降噪算法会影响图像细节吗？

A: 双边滤波是保边降噪算法，在减少噪声的同时尽量保持边缘清晰。

### Q: COMBINED算法处理时间较长，正常吗？

A: 是的，COMBINED组合了多种算法，处理时间较长是正常的。建议对单张图像只使用一次COMBINED。

### Q: 如何获得最佳效果？

A: 建议按以下顺序处理：
1. 降噪（DENOISE）
2. 增强对比度（CLAHE）
3. 锐化（UNSHARP_MASK）
4. 最后微调亮度和对比度
