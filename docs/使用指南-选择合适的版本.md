# 使用指南：选择合适的ImageProcessor版本

## 📋 快速决策树

```
开始
  ↓
需要高质量专业天文图像处理？
  ├─ 是 → 是否能接受~200MB的JAR文件？
  │       ├─ 是 → 使用 OpenCV版本 ⭐⭐⭐⭐⭐
  │       └─ 否 → 使用 纯Java版本 ⭐⭐⭐
  └─ 否 → 使用 纯Java版本 ⭐⭐⭐⭐⭐
```

## 🎯 版本选择建议

### 场景1：快速开发和测试
**推荐版本**：纯Java版本
- ✅ 无需安装额外依赖
- ✅ 编译速度快
- ✅ JAR文件小，易于分享
- ⚠️ 性能和质量稍逊

```bash
# 直接运行
mvn clean package
java -jar target/astro-observer-1.0.0.jar
```

### 场景2：专业天文观测
**推荐版本**：OpenCV版本
- ✅ 更高质量的图像处理
- ✅ 更高的性能（3-5倍）
- ✅ 图像配准功能
- ✅ 自动白平衡和降噪
- ⚠️ JAR文件较大（~200MB）

```bash
# 已配置OpenCV依赖，直接运行
mvn clean package
java -jar target/astro-observer-1.0.0.jar
```

### 场景3：教学和学习
**推荐版本**：纯Java版本
- ✅ 代码更易理解
- ✅ 无需关心OpenCV的复杂性
- ✅ 专注于算法逻辑

### 场景4：生产部署（远程服务器）
**推荐版本**：根据网络条件选择
- **网络良好**：OpenCV版本（性能优势）
- **网络受限**：纯Java版本（下载快）

## 🔧 如何切换版本

### 从纯Java版本切换到OpenCV版本

1. **修改GUI代码中的import语句**：

找到`AdvancedAstronomyFrame.java`，修改：
```java
// 从这个
import com.astronomy.observer.image.ImageProcessor;
import com.astronomy.observer.image.ImageStacker;

// 改为这个
import com.astronomy.observer.image.OpenCVImageProcessor;
import com.astronomy.observer.image.OpenCVImageStacker;
```

2. **修改实例化代码**：

```java
// 从这个
private final ImageProcessor imageProcessor = new ImageProcessor();
private final ImageStacker imageStacker = new ImageStacker(100);

// 改为这个
private final OpenCVImageProcessor imageProcessor = new OpenCVImageProcessor();
private final OpenCVImageStacker imageStacker = new OpenCVImageStacker(100);
```

3. **重新编译**：
```bash
mvn clean package
```

### 从OpenCV版本切换到纯Java版本

反向操作上述步骤即可。

## 📦 依赖配置

### 纯Java版本（无OpenCV）

```xml
<dependencies>
    <dependency>
        <groupId>com.github.sarxos</groupId>
        <artifactId>webcam-capture</artifactId>
        <version>0.3.12</version>
    </dependency>
    <!-- 其他依赖... -->
</dependencies>
```

### OpenCV版本（当前配置）

```xml
<dependencies>
    <!-- OpenCV - 使用最新的 platform 版本，包含所有本地库 -->
    <dependency>
        <groupId>org.bytedeco</groupId>
        <artifactId>opencv-platform</artifactId>
        <version>4.8.0-1.5.9</version>
    </dependency>

    <dependency>
        <groupId>com.github.sarxos</groupId>
        <artifactId>webcam-capture</artifactId>
        <version>0.3.12</version>
    </dependency>
    <!-- 其他依赖... -->
</dependencies>
```

### 可选：减小OpenCV JAR体积

如果只在一个平台上运行（如Windows），可以使用平台特定依赖：

```xml
<dependency>
    <groupId>org.bytedeco</groupId>
    <artifactId>opencv</artifactId>
    <version>4.8.0-1.5.9</version>
</dependency>
<dependency>
    <groupId>org.bytedeco</groupId>
    <artifactId>opencv-windows-x86_64</artifactId>
    <version>4.8.0-1.5.9</version>
</dependency>
```

这会将JAR从~200MB减少到~50MB。

## 🚀 性能优化建议

### 纯Java版本优化

1. **使用多线程**：
   ```java
   ForkJoinPool pool = new ForkJoinPool(4);
   pool.submit(() -> {
       // 并行处理图像
   }).get();
   ```

2. **缓存结果**：
   ```java
   private Map<String, BufferedImage> cache = new HashMap<>();
   ```

3. **降低图像分辨率**用于预览

### OpenCV版本优化

1. **及时释放资源**：
   ```java
   Mat mat = opencv_imgcodecs.imread("image.png");
   // 使用mat
   mat.release(); // 重要！避免内存泄漏
   ```

2. **使用UMat**（OpenCL加速）：
   ```java
   UMat umat = new UMat();
   // 处理umat
   ```

3. **批量处理图像**而非逐张处理

## 🐛 常见问题

### 纯Java版本

**Q: 处理大图像很慢怎么办？**
A: 降低分辨率，或切换到OpenCV版本

**Q: 检测到的星点不准确？**
A: 调整阈值参数，或使用OpenCV版本的高级算法

### OpenCV版本

**Q: 启动报错 "java.lang.UnsatisfiedLinkError"**
A: 确保OpenCV本地库已正确加载，检查Java版本兼容性

**Q: 内存占用过高？**
A: 及时调用`Mat.release()`释放资源

**Q: 不同平台之间移植困难？**
A: 使用`opencv-platform`依赖，它包含所有平台的本地库

## 📝 总结

选择合适的版本取决于：
1. **性能需求**：OpenCV版本明显更快
2. **质量要求**：OpenCV版本提供专业级算法
3. **部署环境**：网络、存储、平台兼容性
4. **用户群体**：普通用户 vs 专业用户

建议提供两个版本的JAR文件，让用户自行选择。
